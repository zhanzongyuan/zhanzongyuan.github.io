虚拟机搭建私有云

Posted on 2018-09-19 | Edited on 2018-09-20 | In 计算机科学
虚拟机搭建私有云

本人搭建了自己本地的虚拟机作为云服务器，在校园网环境下实现：远程终端登录私有云、远程终端登录云桌面。

本文集合了个人的一些学习见解，同时为有意搭建ss私有云的同学提供参考。

一、前言

云计算提出的愿景，是要像用水用电那样使用IT服务。—知乎某网友

下面是本次教程涉及的一些核心概念

云计算：云计算是一种基于互联网的计算，通过这种方式，共享的软硬件资源、信息、高级服务可以随时随地按需提供给计算机终端和其他设备。云计算描述了一种基于互联网的新的IT服务的增加、使用和交付模式（wiki云计算）

云计算服务模式：1. SaaS，软件即服务。2. PaaS，平台即服务。3. IaaS，架构即服务

云计算部署模型：公有云（各种阿里云、腾讯云）、私有云（公司内部自建管理的IT基础设施）、混合云

云计算核心技术：集群技术、云平台管理技术、分布式数据库技术、数据管理技术


接下来在教程中搭建的个人服务器属于私有云的范畴，原因是服务器基础设施都在我个人配置管理下。该服务器为今后的服务计算实验提供设备基础（不用买服务器！），唯一限制是只能在校园网环境下运行（介于校园网网关的NAT映射没法改）

二、虚拟机搭建私有云

2.1 目标网络结构

下面是我们预期的网络结构，即搭建私有云的网络配置都是参照这个网络图进行的（需要一定的计网实验知识）

  • wifi路由器：这个是我宿舍的路由器，使用校园网DHCP分配IP172.xx.56.200，同时建立了一个WLAN局域网，既作为WLAN局域网网关，又为wifi局域网下的设备动态分配子网IP（通过路由器自带的配置主页可以查询到）
  • 宿主机：这就是使用我路由器wifi的电脑，无线适配器（无线网卡）连接WLAN后得到分配的子网IP192.168.199.201
  • 子网组成（部分子网是连通的）：
    • 校园网：172.xx.56.0/255.255.252.0（网络IP/子网掩码）
      • 接入互联网
    • WLAN局域网：192.168.199.0/255.255.255.0
      • 接入校园网
    • 虚拟机NAT网卡的子网：10.0.2.0/255.255.255.0
      • 接入WLAN网络，是虚拟机与宿主机共用WLAN的网络流量，使得虚拟机可以成功联网
    • 虚拟机Host-Only网卡的子网：192.168.56.0/255.255.255.0
      • 仅用于在宿主机可以ping通虚拟机，因为NAT网卡提供的连接只能由内到外（通过端口映射），在宿主机角度是看不到也不能穿透NAT连接虚拟机的10.0.2.0这个子网。所以宿主机连通虚拟机的方式就有两种：
        • 通过NAT端口转发，将虚拟机22号端口映射到192.168.199.201的22端口，使得宿主机可以直接用ssh连接192.168.199.201以连接虚拟机
        • 通过Host-Only网卡，Host-Only网卡构建了一个新的子网，使得宿主机和虚拟机在同一子网下，使得直接通过192.168.56.2这个IP登录。所以这里的Host-Only网卡就是用于实验宿主机登录虚拟机。

2.2 搭建虚拟机

原料：Windows10、VirtualBox、ubuntu16.04.4.ios

不喜欢Ubuntu的同学可以用其他系统，如CentOS、Debian等

不喜欢VirtualBox的同学可以用VMware，网络配置的要求是一样的，只是可能设置的位置不一样

部分参考pml老师教程

  1. 安装VirtualBox（直接下载官网安装包按指示安装即可）

  2. 配置VirtualBox网络选项

    1. 创建虚拟机内部虚拟网络，使得 Vbox 内部虚拟机可以通过它，实现虚拟机之间、虚拟机与主机的通讯

      • VirtualBox菜单 ：管理 -> 主机网络管理器，创建一块虚拟网卡（可以使用默认的虚拟网卡），网址分配：192.168.56.1/24（这就是后续会用于Host-Only的虚拟网卡）

      • 在主机 windows 命令行窗口输入 ipconfig 就可以看到 VirtualBox Host-Only Network #?: 的网卡

  3. 安装虚拟机

    • 下载 Linux 发行版镜像
      • 如果是 Centos，仅需要 Minimal ISO；如果是 Ubuntu 请下载桌面和服务器
      • 阿里云OPSX 下载
    • 用 VBox 创建虚拟机。 虚拟机名称建议以 centos-xxx 或 ub-xxx 命名，如果向导不能创建 64 bit 虚拟机，请更换电脑!!!
      • 建议虚拟机CPU、内存采用默认。如果是桌面版，CPU建议数1-2，内存不低于2G
      • 显示，显存采用默认。如果是桌面版，显存越大越好
      • 存储，不低于30G。避免以后扩展难。
      • 网络，第一块网卡必须是 NAT；第二块网卡连接方式： Host-Only，接口就是前面创建的虚拟网卡
    • VirtualBox还提供复制功能，可以将虚拟机链接复制，这样由一个主虚拟机链接复制多个副本，免去重新配置的麻烦（复制时注意选择重新初始化所有网卡的 MAC 地址）

  4. nmtui配置虚拟机网卡

    • 先ifconfig查看网卡对应的IP，这时会有两个网卡，一个是NAT的网卡，一个是Host-Only的网卡。此时应该已经能够上网（测试：ping baidu.com）
    • nmtui进入编辑Host-Only网卡的IP地址（换成你喜欢的地址，确保保证IP段如：192.168.56.xxx/24）。这一步不要配置网关gw！！！配置了的话会导致默认网关为192.168.56.1导致不会用NAT网卡的默认网关联网，直接导致不能联网！（这一步真的很坑！配置完gw的我折腾了好久才发现网关导致不能联网）。
    • Host_Only的网卡确定后，就能在宿主机ping虚拟机的Host-Only IP（ping不通可能会需要打开防火墙ipv4-in），甚至在git bash上SSH登录虚拟机（需要sudo apt install openssh-server）

2.3 远程桌面登录、终端登录

  1. 配置VirtualBox远程桌面、NAT网卡转发规则（这步可以实现在WLAN局域网内登录云桌面和SSH登录终端）
    • 在VirtualBox的虚拟机设置里面（设置-》显示-》远程桌面）可以配置该虚拟机的远程桌面功能。设置端口号（默认3389，如果宿主机开了远程桌面，3389端口就不能正确连接虚拟机桌面，只会连接到宿主机桌面，所以要换一个端口，如：3390）
    • 在VirtualBox的虚拟机设置里配置1号NAT网卡的转发规则：10.0.2.15:22->192.168.199.201:22，通过该配置实现在WLAN局域网下可以直接SSH连接192.168.199.201端口的虚拟机终端。


  1. 配置路由器NAT端口转发规则 or DMZ主机
    • 【推荐】NAT端口转发规则，按照需要按照需要对外开放宿主机的IP端口，如：保证SSH连接的22端口、远程桌面的3389端口。上述虚拟机在宿主机上NAT转发的SSH登录端口号和VBox配置的远程桌面端口只有在通过路由器的端口转发配置，才能在校园网网络里成功SSH终端登录和云桌面登录。
    • 【不推荐】DMZ(Demilitarized Zone)即俗称的非军事区。路由器的DMZ主机指的是WLAN网络中不设防的区域，即将外网对路由器172.xx.56.200的访问都指向DMZ主机，将DMZ主机对外网开放。所以在路由器配置页面上配置DMZ主机为宿主机IP192.168.199.201，则宿主机的所有端口都会在校园网通过IP172.xx.56.200开放。全部端口对外开放其实是个不太安全的设置。


配置完NAT转发规则后，你就可以在校园网环境下用IP172.xx.56.200自由SSH登录虚拟机终端和云桌面（用windows远程桌面，或者其他支持RDP协议的云桌面软件，注意正确设置远程桌面端口）

2.4 常见问题汇总

  1. 虚拟机ping不通宿主机192.168.56.1：打开防火墙ipv4-in
  2. nmtui配置完，ifconfig没有显示成功配置好的ip结果：进入nmtui，重新启动网卡
  3. 宿主机无法ssh登录虚拟机（报错connect fuse）：检查虚拟机是否开启ssh服务ps -ef | grep ssh，参考linux开启SSH服务
  4. 外网无法SSH登录虚拟机or无法登录虚拟机远程桌面：配置路由器端口转发规则，或配置DMZ主机
  5. 虚拟机远程桌面总是连上宿主机：修改VirtualBox远程桌面端口默认端口3389为其他（如：3390），原因是宿主机远程桌面服务端口占用了虚拟机远程桌面端口
  6. 配置完Host-Only网卡后无法联网：检查路由表route，发现默认网关为192.168.56.1，原因是配置nmtui的时候给第二款网卡配置了默认gw，导致默认网关修改，使得NAT网关与Host-Only网关冲突。进入nmtui将Host-Only网卡的gw清空即可。
